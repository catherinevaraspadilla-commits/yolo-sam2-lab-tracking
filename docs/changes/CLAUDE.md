# Changes

Observation log for tracking pipeline improvements over time.

Each entry documents what was observed in a video run, what the problem was,
what was changed, and whether it improved. This creates a traceable history
of iterative improvements.

## Files

- `observation_log.md` — Chronological log of observations, problems, and fixes
- `pipeline_comparison.md` — Side-by-side comparison of pipeline runs on the same video
